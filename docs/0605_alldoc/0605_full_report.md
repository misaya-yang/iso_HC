# IsoHC 实验综合报告

**日期**: 2026-05-11  
**硬件**: NVIDIA RTX 5090 (32GB)  
**环境**: PyTorch 2.8.0+cu128, Python 3.12.3

---

## 1. 执行摘要

本报告汇总了 IsoHC (Isotropic Hypercube Connectivity) 从 Stage 1 到 Stage 2 的全部实验结果，涵盖核心数学验证、深层稳定性测试、真实数据训练、GNN 场景验证和极限深度对照实验。

**核心结论**: IsoHC 的 fixed-vector isometric projection 在所有测试场景下均表现出完美的能量保持和数值稳定性，在 1024 层深度下仍保持 grad=1.0000、energy=1.0000，而对照方法（unconstrained、mhc-lite、gcn-diffusion）分别出现爆炸、收缩或 collapse。

## 6. 关键发现与结论

### 6.1 数学正确性
- SVD-based polar decomposition 在 n≤16 时精度 ~1e-6
- NS K=5 对 n≥8 收敛到 SVD 级别精度
- Fix error（固定向量保持）始终 < 1e-6

### 6.2 深层稳定性
- **IsoHC 在 1024 层仍保持 grad=1.0, energy=1.0**
- unconstrained 在 256 层爆炸到 74M 倍，512 层到 6.8e+10 倍
- mhc-lite 能量收缩到 ~25-50%，且 cosine collapse
- gcn-diffusion 同样收缩到 ~25-35%

### 6.3 数值精度
- **bf16_all 不可行**: mean_err 高达 70%
- **bf16_fp32_mix 可行**: 投影在 fp32 中执行，其余用 bf16

### 6.4 GNN 场景
- IsoHC 成功避免了 GNN oversmoothing
- 但 IsoStream-GNN architecture 需要进一步优化 accuracy
- 核心机制已验证，architecture 设计是下一步重点

### 6.5 训练稳定性
- 16-64 层 Transformer 稳定训练，无 NaN
- Projection error 在训练过程中不漂移
- 训练开销 ~35%（主要来自 SVD 投影）

## 7. 下一步建议

1. **优化 IsoStream-GNN architecture**: 调整 β、初始化、grad clip 以提高 downstream accuracy
2. **更大规模真实数据**: WikiText-103、C4 等
3. **mHC 正式对照**: 完整的 mHC 实现和对比
4. **梯度分析**: stream diversity、gradient profile 诊断
5. **效率优化**: 探索更快的 polar decomposition 近似（如 3-5 步 NS）
